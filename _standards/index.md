            <article class="docs-method">
                <a class="docs-method-target" id="/check/init/"  name="/check/init/"></a>
                    <h3 class="docs-method-title">
                        <a href='#/check/init/'>初始化</a>
                    </h3>
                    <a class="docs-method-edit" href="javascript:;" target="_blank">编辑</a>
                <div class="docs-method-prose">
                    <p>此插件是基于jquery的，初始化的时候，我们需要引入jquery（本系统已内置）或者zepto，然后再引入icheck.js（已内置）</p>
                    <p>iCheck支持所有选择器（selectors），并且只针对复选框和单选按钮起作用：</p>
<pre>
&lt;script>
// customize all inputs (will search for checkboxes and radio buttons)
$('input').iCheck();

// handle inputs only inside $('.block')
$('.block input').iCheck();

// handle only checkboxes inside $('.test')
$('.test input').iCheck({
  handle: 'checkbox'
});

// handle .vote class elements (will search inside the element, if it's not an input)
$('.vote').iCheck();

// you can also change options after inputs are customized
$('input.some').iCheck({
  // different options
});

&lt;/script>
</pre>
                </div>
            </article>
            <article class="docs-method">
                <a class="docs-method-target" id="/check/callback/"  name="/check/callback/"></a>
                    <h3 class="docs-method-title">
                        <a href='#/check/callback/'>回调事件</a>
                    </h3>
                    <a class="docs-method-edit" href="javascript:;" target="_blank">编辑</a>
                <div class="docs-method-prose">
                    <p>iCheck提供了大量回调事件，都可以用来监听change事件。</p>
                    <p>
                        <ul class="check-event-ul">
                            <li><a>ifClicked</a><span>用户点击了自定义的输入框或与其相关联的label</span></li>
                            <li><a>ifChanged</a><span>输入框的 checked 或 disabled 状态改变了</span></li>
                            <li><a>ifChecked</a><span>输入框的状态变为 checked</span></li>
                            <li><a>ifUnchecked</a><span>checked 状态被移除</span></li>
                            <li><a>ifDisabled</a><span>输入框状态变为 disabled</span></li>
                            <li><a>ifEnabled</a><span>disabled 状态被移除</span></li>
                            <li><a>ifCreated</a><span>输入框被应用了iCheck样式</span></li>
                            <li><a>ifDestroyed</a><span>iCheck样式被移除</span></li>
                        </ul>
                    </p>
                    <p>使用 on() 来绑定回调事件</p>
<pre>
&lt;script>
// bind the ifChecked event 
$('input').on('ifChecked', function(event){
  alert(event.type + ' callback');
});
&lt;/script>
</pre>
                    <p>ifCreated 事件很特殊，在 iCheck 插件初始化之前绑定</p>
                </div>
            </article>
            <article class="docs-method">
                <a class="docs-method-target" id="/check/function/"  name="/check/function/"></a>
                    <h3 class="docs-method-title">
                        <a href='#/check/function/'>方法</a>
                    </h3>
                    <a class="docs-method-edit" href="javascript:;" target="_blank">编辑</a>
                <div class="docs-method-prose">
                    <p>下面这些方法可以用来通过编程方式改变输入框状态（可以使用任何选择器）：</p>
<pre>
&lt;script>
$('input').iCheck('check'); // 将输入框的状态设置为checked
$('input').iCheck('uncheck'); // 移除 checked 状态
$('input').iCheck('toggle'); // toggle checked state
$('input').iCheck('disable'); // 将输入框的状态设置为 disabled
$('input').iCheck('enable'); // 移除 disabled 状态
$('input').iCheck('update'); // apply input changes, which were done outside the plugin
$('input').iCheck('destroy'); // 移除iCheck样式
&lt;/script>
</pre>
                </div>
